# Plan: Enable Offline Functionality for SentinelOps

## Current State Analysis

**What Already Exists:**
- `offline-start.ps1` script - launches all services locally (server, ML, client)
- Environment variables: `OFFLINE_MODE`, `NO_CLOUD_MODE`
- Zustand persist middleware for auth in localStorage
- Basic API URL resolution logic in api.js

**What's Missing:**
- No PWA (Progressive Web App) configuration
- No Service Worker for asset caching
- No Web App Manifest
- No offline API fallback/queue
- No IndexedDB for data persistence
- No network status detection UI

---

## Features Added

### 1. PWA Configuration (COMPLETED)
- Added `vite-plugin-pwa` to cache static assets
- Created Web App Manifest with icons (`manifest.webmanifest`)
- Enabled install prompt handling (PwaInstallPrompt component)

### 2. Service Worker Strategies (COMPLETED)
- **Cache First** for static assets (JS, CSS, images, fonts)
- **Network First** for API calls with offline fallback

### 3. IndexedDB Data Layer (COMPLETED)
- Created IndexedDB service using `idb` library
- Stores: datasets, anomalies, auditLogs, syncQueue, cache
- Background sync when online

### 4. Network Status Detection (COMPLETED)
- Added `useNetworkStatus` hook with online/offline detection
- Created `OfflineIndicator` UI component
- Shows pending sync operations count
